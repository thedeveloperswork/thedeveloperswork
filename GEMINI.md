- **`02_experience.md`**: Quantified professional achievements and enterprise roles.

## 🧠 Global Operational Rules
These rules apply universally to ALL outputs generated by the agent, regardless of the workflow executed:

1. **The XYZ Formula (Impact Quantification):** Every generated bullet point must strictly adhere to: "Accomplished [X] as measured by [Y], by doing [Z]".
    - Examples: "Slashed execution time by 60%..." or "Achieved a 0% defect injection rate...". 
    - Eliminate subjective fluff; rely purely on technical metrics (%, time, TB, count).
2. **Strict Chronology:** Always order Projects, Experience, and Education in strictly descending order based on their timelines (most recent first).
3. **Linguistic Diversity:** Do not repeat the same action verb more than twice in any single generated document. Synthesize technical synonyms (e.g., Architected, Engineered, Orchestrated, Pioneered).

## 🚀 CI/CD Management
All deployments are automated via GitHub Actions and are **only** triggered when a Pull Request is successfully merged into the target branch. A mandatory **Validate PR** guardrail runs on all open Pull Requests.

1. **Guardrail (`validate-testing.yml`)**
   - **Trigger**: Pull Request opened to `testing`.
   - **Checks**: Linting (errors only) and Vitest unit tests.
   - **Goal**: Prevent broken or poorly typed code from being merged.

2. **Automated Deployment (`deploy-portfolio.yml`)**
   - **Trigger**: `push` to `master` or `testing`.
   - **Environment**: 
     - `github-pages` for Production (`master`).
     - `alpha` for Testing (`testing`).
   - **Strategy**: 
     - **Production**: Official GitHub Pages API deployment (artifact-based).
     - **Testing**: Branch-based deployment to `gh-pages-alpha`.
   - **Linting**: Strict (`--max-warnings 0`) for `master`; standard for `testing`.

3. **Manual Development Deployment (`deploy-portfolio-dev.yml`)**
   - **Trigger**: `workflow_dispatch` (Manual only).
   - **Target**: Branch-based deployment to `gh-pages-dev`.
   - **Purpose**: Ad-hoc testing from any experimental branch.




4. **Resume Pipeline**
   - **Status**: Manual generation. PDF compilation is removed from the automated CI/CD pipeline and should be handled locally using `pdflatex` to ensure exact tailoring before commit.
